# 一文（いちもん） — ikesu-capture

> 残った一文と、その理由。

読書中に「印象に残った一文 → なぜ印象に残ったか → 自分の領域への示唆」を摩擦ゼロで残し、
それを **情報のイケス**（`~/work/ikesu/10_books/`）に貯めるためのキャプチャツール。

NotebookLM で本と対話し、その学びをここに貯める。AIの対話機能は持たない（NotebookLMに任せる）。
デザインは「藍墨」— 濃紺 × 琥珀のダークテーマ。

## 開く

**web版（どこからでも・サーバー起動不要）**：
https://contact329.github.io/ichimon/

- https なので Mac の Chrome / Edge なら「10_books に直接保存」もそのまま使える
- スマホからも開いて記録できる（直書きは不可 → コピー / .mdをDL）
- ⚠️ データ（localStorage）は **URLごと** に別管理。web版と下記ローカル版でデータは共有されない。Mac↔スマホの同期もしない（同期はバックエンドが必要な別フェーズ）
- ホスティング元: GitHub `contact329/ichimon`（main ブランチ root を GitHub Pages 配信）。更新は `git push` で自動反映

## 起動（ローカル版）

**ダブルクリックで起動**（推奨）：

```
一文をひらく.command
```

→ サーバーが未起動なら自動で立ち上げ、ブラウザで http://localhost:4177 が開く。

（手動で起動する場合）

```bash
python3 -m http.server 4177 --directory ~/work/tools/ikesu-capture
```

> File System Access API（直書き）は localhost / https でのみ動くため、`index.html` を
> `file://` で直接開くと「直接保存」が使えない。必ずサーバー経由（localhost）で開くこと。

## 使い方

1. **＋新しい本** で本を登録（書名・著者・NotebookLMのURL）
2. **NotebookLM でリサーチ**：
   - ①②③（解説・書評・批判 / 著者の声・論争 / 実務での応用事例）を **NotebookLM の「ソースを探す」** に貼ってソースを集める（タイトルは自動挿入）
   - 集めたソース＋本のPDFを入れたら、**「ソクラテス式の対話」** プロンプトでチャット深掘り
   - ①②③は Gemini / Perplexity でも使える収集クエリ
3. **一文を残す**：読みながら
   - 印象に残った一文（＊必須）
   - なぜ印象に残ったか（＊必須・一番大事）
   - 自分の領域への示唆（任意・Willbe / YouTube / 事業 …）
   - 概念タグ（任意・`[[ ]]` で囲われる。既出タグはチップをクリックで再利用＝表記揺れ防止）
   - `Cmd/Ctrl + Enter` でも保存できる
4. **イケスに書き出す**：溜まったカードを `書名.md`（イケス形式）に変換

## イケスへの保存（3経路）

書き出しモーダルから選べる：

| ボタン | 内容 |
|---|---|
| 10_books に直接保存 | **推奨**。初回だけ `~/work/ikesu/10_books` フォルダを選択 → 以降ワンクリックで直書き（Chrome / Edge） |
| .md をDL | `書名.md` をダウンロード → 手動で `10_books/` に置く |
| コピー | Markdown をコピー → Obsidian に貼る |

## データ保存

- 入力中のハイライト・本のリストは **このブラウザの localStorage**
- 「直接保存」用のフォルダハンドルは IndexedDB に保持（次回の選択が不要に）
- 最終成果物は `~/work/ikesu/10_books/書名.md`
